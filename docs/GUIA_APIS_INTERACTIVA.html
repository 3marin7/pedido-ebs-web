❓¿Qué es una API?

API = Interfaz de Programación de Aplicaciones

Es un "puente" que permite que dos aplicaciones se comuniquen entre sí. Imagina que es como un mesero en un restaurante: tú (cliente) le pides algo al mesero (API), y él va a la cocina (servidor) a traer lo que ordenaste.
Tu App (Frontend)
→
API
→
Servidor/Servicio

🔄¿Cómo funciona?

  1. Tu app hace una solicitud (pide datos)
  2. La API recibe la solicitud (entiende qué pides)
  3. El servidor procesa (busca en la base de datos)
  4. La API devuelve la respuesta (datos en formato JSON)
  5. Tu app recibe los datos (mostrados en pantalla)

📊Tipos de APIs usadas en EBS

En el proyecto EBS usamos principalmente dos tipos:
Tipo Descripción Ejemplo en EBS
REST Usa URLs y métodos HTTP (GET, POST, PUT, DELETE) Supabase: obtener clientes, crear facturas
URL Scheme Abre una app o servicio mediante URL especial WhatsApp: abrir chat con número
CDN Red de distribución de contenido (archivos estáticos) Google Fonts, Font Awesome

🛠️Las 4 APIs del Proyecto EBS

🗄️1. SUPABASE (Base de Datos)

REST API Real-time

¿Qué hace? Almacena y gestiona todos los datos: clientes, facturas, productos, abonos, etc.

🔗 URL: https://jqkfykverasqwlfsjsnj.supabase.co

📁 Archivo principal: src/lib/supabaseApi.js

// Ejemplo: Obtener todos los clientes async obtenerClientes() { const { data, error } = await supabase .from('clientes') .select('*') .eq('activo', true) .order('nombre') return data }
En la app: ¿Dónde se usa?
  • ✔ Cargar lista de clientes en "Crear Factura"
  • ✔ Guardar nueva factura cuando haces clic en "Guardar"
  • ✔ Mostrar productos en el catálogo
  • ✔ Registrar movimientos de inventario
Operaciones principales:
Operación Método HTTP Qué hace
select() GET Lee datos (⬇️ traer)
insert() POST Crea datos (➕ nuevo)
update() PUT Modifica datos (✏️ editar)
delete() DELETE Elimina datos (🗑️ borrar)

✅ Ventajas

  • Rápida y confiable
  • Fácil de usar con JavaScript
  • Actualizaciones en tiempo real
  • Controla quién ve qué (RLS)
  • Backup automático

❌ Limitaciones

  • Requiere cuenta y API key
  • Limitado por plan gratuito
  • Depende de internet
  • Si Supabase cae, no funciona

🖼️2. CLOUDINARY (Subida de Imágenes)

REST API

¿Qué hace? Recibe imágenes de tu computadora y las guarda en la nube.

🔗 URL: https://api.cloudinary.com/v1_1/dstnroimw/image/upload

📁 Archivo: src/components/CatalogoProductos.jsx (línea 32)

const formData = new FormData() formData.append('file', file) formData.append('upload_preset', 'catalogo_productos_web') const response = await fetch( 'https://api.cloudinary.com/v1_1/dstnroimw/image/upload', { method: 'POST', body: formData } )
En la app: ¿Dónde se usa?
  • ✔ Botón "📤 Subir Imagen" en Catálogo de Productos
  • ✔ Carga la imagen a la nube
  • ✔ Devuelve URL para guardar en Supabase
Flujo:
Usuario selecciona imagen
→
Se envía a Cloudinary
→
Guarda en nube
→
Devuelve URL

✅ Ventajas

  • Comprime automáticas
  • Muy rápido (CDN global)
  • Optimiza para mobile
  • Almacenamiento ilimitado

❌ Limitaciones

  • Plan gratuito limitado
  • Requiere API key
  • Dependencia de servicio externo

💬3. WHATSAPP (Envío de Mensajes)

URL Scheme

¿Qué hace? Abre WhatsApp con un mensaje predefinido para enviar pedidos.

🔗 URLs:

  • https://api.whatsapp.com/send (Web)
  • https://wa.me/ (Directo)

📁 Archivos:

  • src/components/CatalogoClientes.jsx (línea 420)
  • src/components/FacturaDetalle.jsx (línea 853)
// Generar URL para WhatsApp const numero = '573004583117' const mensaje = '¡Hola! Quiero hacer un pedido...' const url = `https://wa.me/${numero}?text=${encodeURIComponent(mensaje)}` window.open(url)
En la app: ¿Dónde se usa?
  • ✔ Botón "📤 Enviar por WhatsApp" en el carrito
  • ✔ Botón "Compartir factura" después de guardar
  • ✔ Abre WhatsApp automáticamente con el pedido
Flujo:
Usuario hace clic "Enviar por WhatsApp"
→
App genera URL con mensaje
→
Abre WhatsApp
→
Mensaje prefilled

✅ Ventajas

  • Gratis (no requiere API)
  • Muy directo
  • Personalizable
  • Funciona sin cuenta

❌ Limitaciones

  • Requiere que usuario tenga WhatsApp
  • No verifica si se envió
  • Límites de velocidad

⚡4. CDN (Recursos Externos)

Content Delivery Network

¿Qué hace? Distribuye archivos (fuentes, iconos) desde servidores rápidos en todo el mundo.

📁 Archivos CSS del proyecto:

/* Google Fonts */ @import url('https://fonts.googleapis.com/css2?family=Roboto') /* Font Awesome (iconos) */ @import url('https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.0.0/css/all.min.css')
En la app: ¿Dónde se usa?
  • ✔ Fuente Roboto en títulos y texto
  • ✔ Iconos Font Awesome (⬇️ 📦 🔍 etc)
  • ✔ Estilos visuales de la aplicación
Servicio Propósito URL
Google Fonts Tipografías fonts.googleapis.com
Font Awesome Iconos cdnjs.cloudflare.com
vía.placeholder.com Imágenes placeholder via.placeholder.com

✅ Ventajas

  • Muy rápido globalmente
  • No ocupan espacio local
  • Actualizaciones automáticas
  • Caché en navegador

❌ Limitaciones

  • Requiere internet
  • Si CDN cae, no se cargan
  • No puedes customizar

🔄Flujo Completo: De usuario a API

Veamos cómo interactúan todas las APIs cuando un usuario crea y envía un pedido:
Paso 1️⃣: Usuario carga la app → App solicita productos a SUPABASE con SELECT → Carga todos los productos en pantalla
Paso 2️⃣: Usuario quiere subir imagen de producto → Click en "📤 Subir Imagen" → App envía imagen a CLOUDINARY → Cloudinary devuelve URL de la imagen → App guarda URL en SUPABASE
Paso 3️⃣: Usuario arma pedido y quiere enviarlo → Click en "💬 Enviar por WhatsApp" → App genera URL con todos los datos → Abre WHATSAPP con mensaje prefilled → Usuario recibe pedido formateado
Paso 4️⃣: App muestra estilos bonitos → Carga fuentes de GOOGLE FONTS (CDN) → Carga iconos de FONT AWESOME (CDN) → App se ve profesional y rápido

🎯Puntos Clave a Recordar

✅ API Exitosa

  • Devuelve datos correctamente
  • Respuesta rápida
  • Formato JSON válido
  • Sin errores 500

❌ API Fallida

  • Timeout (demora excesiva)
  • Error 404 (no encontrado)
  • Error 401 (sin permiso)
  • Respuesta malformada
⚠️ Recuerda:
  • Sin internet = Sin APIs (siempre necesitan conexión)
  • URLs y credenciales deben guardarse en .env, nunca visible en código
  • Documenta cada API: qué hace, URL, autenticación
  • Maneja errores siempre con try/catch
  • Las APIs son el "puente" entre frontend y datos

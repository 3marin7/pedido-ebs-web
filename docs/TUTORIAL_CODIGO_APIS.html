🗄️ SUPABASE: Operaciones CRUD

¿Qué significa CRUD?

Create (Crear) • Read (Leer) • Update (Actualizar) • Delete (Eliminar)

1️⃣ READ - Obtener Clientes (SELECT)

Objetivo: Traer todos los clientes activos desde la base de datos

1 // Archivo: src/lib/supabaseApi.js
2
3 async obtenerClientes() {
4 try {
5 const { data, error } = await supabase
6 .from('clientes') // ← Tabla a consultar
7 .select('*') // ← Seleccionar todas las columnas
8 .eq('activo', true) // ← Filtro: solo activos
9 .order('nombre') // ← Ordenar alfabéticamente
10
11 if (error) throw error // ← Si hay error, lanzarlo
12 return data // ← Devolver los datos
13 } catch (error) {
14 console.error('Error:', error)
15 throw error
16 }
17 }
🔍 Explicación línea por línea:

Línea 3: async = esperará respuesta de API | function obtenerClientes() = nombre de la función

Línea 5: await supabase = espera a que Supabase responda

Línea 6: .from('clientes') = accede a la tabla "clientes"

Línea 7: .select('*') = trae todas las columnas (el * = "todas")

Línea 8: .eq('activo', true) = solo clientes donde activo = true

Línea 9: .order('nombre') = ordena alfabéticamente por nombre

Línea 11: if (error) throw error = si falla, lanza el error

Línea 12: return data = devuelve los datos al componente

🔄 Flujo de ejecución: App llama: await obtenerClientes()
↓ Envía solicitud HTTP GET
↓ Supabase busca en tabla "clientes"
↓ Filtra por activo = true
↓ Ordena por nombre
↓ Devuelve array de clientes
↓ App recibe: [{id: 1, nombre: "Juan", ...}, ...]
📱 Ejemplo real en pantalla: Usuario abre "Crear Factura" → App llama obtenerClientes() → Supabase devuelve lista → Dropdown se llena con nombres de clientes

2️⃣ CREATE - Crear Nueva Factura (INSERT)

Objetivo: Guardar una nueva factura en la base de datos

1 async crearFactura(factura) {
2 try {
3 const { data, error } = await supabase
4 .from('facturas') // ← Tabla destino
5 .insert([{ // ← Insertar nuevo registro
6 cliente_id: factura.cliente_id,
7 fecha: factura.fecha,
8 vendedor_id: factura.vendedor_id,
9 total: parseFloat(factura.total), // ← Convertir a número
10 }])
11 .select() // ← Devolver el registro creado
12
13 if (error) throw error
14 return data[0] // ← Devolver primer registro (el creado)
15 } catch (error) {
16 console.error('Error creando factura:', error)
17 throw error
18 }
19 }
🔍 Explicación:

Línea 5: .insert([{...}]) = crea un nuevo registro EN la tabla

Línea 6-9: Datos a guardar = id del cliente, fecha, vendedor, total

Línea 11: .select() = devuelve el registro recién creado con su ID

Línea 14: data[0] = obtiene el primer (y único) registro del array devuelto

🔄 Flujo: Usuario click "Guardar Factura"
↓ App recolecta datos (cliente_id, fecha, vendedor_id, total)
↓ Llama: await crearFactura({...})
↓ Supabase INSERT en tabla "facturas"
↓ BD genera ID automático (ej: 12345)
↓ Devuelve: {id: 12345, cliente_id: 5, fecha: "2026-04-28", ...}
↓ App muestra "Factura guardada: #12345"

3️⃣ UPDATE - Actualizar Cliente (UPDATE)

Objetivo: Modificar un cliente existente

1 async actualizarCliente(id, cliente) {
2 try {
3 const { data, error } = await supabase
4 .from('clientes')
5 .update(cliente) // ← Los nuevos datos
6 .eq('id', id) // ← Actualizar SOLO donde id = [este id]
7 .select()
8
9 if (error) throw error
10 return data[0]
11 } catch (error) { ... }
12 }
🔍 Puntos clave:

Línea 5: .update(cliente) = el objeto con los nuevos datos

Línea 6: .eq('id', id) = IMPORTANTE: especifica CUÁL actualizar (por ID)

⚠️ Sin el .eq(), actualizarías TODOS los clientes!

4️⃣ DELETE - Eliminar Registro (DELETE)

1 async eliminarFactura(id) {
2 const { error } = await supabase
3 .from('facturas')
4 .delete() // ← Eliminar
5 .eq('id', id) // ← DONDE id = [este]
6 if (error) throw error
7 return 'Eliminado correctamente'
8 }
⚠️ CUIDADO: Especifica siempre .eq() o .where() para no eliminar TODO. Si ejecutas .delete() sin filtro, borrarás toda la tabla.

Resumen: Los 4 CRUD

Operación Método HTTP Código Ejemplo
CREATE POST .insert([...]) Guardar factura nueva
READ GET .select() Cargar lista de clientes
UPDATE PUT .update({...}) Cambiar teléfono de cliente
DELETE DELETE .delete() Borrar factura

🖼️ CLOUDINARY: Subida de Imágenes

¿Cómo funciona?

Cloudinary es un servicio para alojar imágenes en la nube. En lugar de guardar imágenes en tu servidor, las subes a Cloudinary y él devuelve una URL.

Paso 1: Configuración

Primero necesitas:

  1. Crear cuenta en Cloudinary.com
  2. Obtener tu Cloud Name: dstnroimw
  3. Obtener upload_preset: catalogo_productos_web

Paso 2: Código del Upload

Ubicación: src/components/CatalogoProductos.jsx (línea 32)

1 // Componente CloudinaryUpload
2 const handleFileUpload = async (e) => {
3 const file = e.target.files[0] // ← Obtener archivo seleccionado
4 if (!file) return
5
6 // Validación 1: Verificar que es imagen
7 if (!file.type.match('image.*')) {
8 alert('Por favor, selecciona un archivo de imagen')
9 return
10 }
11
12 // Validación 2: Tamaño máximo 5MB
13 if (file.size > 5 * 1024 * 1024) {
14 alert('Imagen muy grande (máximo 5MB)')
15 return
16 }
17
18 // Crear FormData para enviar archivo
19 const formData = new FormData()
20 formData.append('file', file) // ← El archivo
21 formData.append('upload_preset', 'catalogo_productos_web') // ← Credencial
22
23 try {
24 setUploading(true) // ← Mostrar "Subiendo..."
25
26 // ✅ ENVÍO A CLOUDINARY
27 const response = await fetch(
28 'https://api.cloudinary.com/v1_1/dstnroimw/image/upload', // ← URL de Cloudinary
29 {
30 method: 'POST', // ← Método POST
31 body: formData // ← El archivo empaquetado
32 }
33 )
34
35 // Convertir respuesta a JSON
36 const data = await response.json()
37
38 // Devolver URLs de la imagen
39 onImageUpload({
40 imagenUrl: data.secure_url, // ← URL de la imagen guardada
41 imagenPublicId: data.public_id // ← ID para futuras operaciones
42 })
43 } catch (error) {
44 console.error('Error subiendo imagen:', error)
45 alert('Error al subir la imagen')
46 } finally {
47 setUploading(false) // ← Ocultar "Subiendo..."
48 }
49 }
🔍 Desglose:

Línea 3: e.target.files[0] = obtiene el archivo que el usuario seleccionó

Línea 7: file.type.match('image.*') = valida que sea imagen

Línea 13: file.size > 5 * 1024 * 1024 = máximo 5MB

Línea 19: new FormData() = contenedor para enviar archivos

Línea 20-21: formData.append() = agrega datos al contenedor

Línea 27-33: fetch() POST = envía a Cloudinary

Línea 40: data.secure_url = URL HTTPS de la imagen

Línea 41: data.public_id = ID para borrar o modificar después

🔄 Flujo completo: Usuario click input file
↓ Selecciona imagen (ej: "producto.jpg")
↓ handleFileUpload() se ejecuta
↓ Valida: ¿es imagen? ¿menos de 5MB?
↓ Crea FormData con archivo
↓ fetch() POST a Cloudinary
↓ Cloudinary procesa imagen
↓ Devuelve: { secure_url: "https://res.cloudinary.com/...", public_id: "xyz123" }
↓ App guarda URL en Supabase
↓ Imagen aparece en pantalla 🖼️

Paso 3: Guardar en Supabase

Después de uploadear a Cloudinary, guardamos la URL en Supabase:

💬 WHATSAPP: Compartir Pedidos

¿Cómo funciona?

WhatsApp tiene un "URL Scheme" que permite abrir un chat con un mensaje predefinido. No usamos API oficial (eso requiere pagar), usamos el método gratis.

Paso 1: Generar el Mensaje

Ubicación: src/components/CatalogoClientes.jsx (línea 420)

1 const enviarPedidoWhatsApp = async () => {
2 // Validar que hay productos
3 if (productosSeleccionados.length === 0) {
4 alert('Selecciona al menos un producto')
5 return
6 }
7
8 // Número de WhatsApp (sin + ni espacios)
9 const numero = '573002945085' // ← Número destino
10
11 // Comenzar a construir el mensaje
12 let mensaje = `*¡NUEVO PEDIDO!*%0A%0A` // ← %0A = salto de línea
13 mensaje += `*Cliente:* ${clienteInfo.nombre}%0A`
14 mensaje += `*Teléfono:* ${clienteInfo.telefono}%0A`
15
16 // Agregar productos al mensaje
17 mensaje += `%0A*📦 PRODUCTOS:*%0A%0A`
18 productosSeleccionados.forEach(p => {
19 mensaje += `✔️ ${p.nombre}%0A`
20 mensaje += ` Cantidad: ${p.cantidad}%0A`
21 mensaje += ` Precio: $${p.precio}%0A`
22 mensaje += ` Subtotal: $${p.precio * p.cantidad}%0A%0A`
23 })
24
25 // Total
26 mensaje += `*TOTAL: $${calcularTotal()}*`
27
28 // Codificar mensaje para URL
29 const mensajeCodificado = encodeURIComponent(mensaje)
30
31 // Generar URL de WhatsApp
32 const url = `https://wa.me/${numero}?text=${mensajeCodificado}`
33
34 // Abrir WhatsApp
35 window.open(url)
36 }
🔍 Desglose:

Línea 9: Número sin + ni espacios = 573002945085 (no +57 +57 30029)

Línea 12: %0A = salto de línea | * = negrita en WhatsApp

Línea 18-23: .forEach() = itera productos y agrega cada uno al mensaje

Línea 29: encodeURIComponent() = convierte caracteres especiales a URL-safe

Línea 32: Template string = construye URL de WhatsApp

Línea 35: window.open() = abre URL en ventana nueva

⚠️ Importante: El mensaje usa %0A (codificado) internamente pero encodeURIComponent lo convierte automáticamente. Los saltos de línea se ven correctamente en WhatsApp.

Paso 2: URL de WhatsApp

La URL generada se ve así (aunque larga y codificada):

1 https://wa.me/573002945085?text=*%21NUEVO%20PEDIDO%21*%0A%0A*Cliente*%3A%20Juan%0A*Tel%C3%A9fono*%3A%203004583117%0A...
Decodificado se ve así: https://wa.me/573002945085?text=*¡NUEVO PEDIDO!*%0ACli ente: Juan%0ATeléfono: 3004583117...

💡 Tips:
  • Funciona en cualquier dispositivo (web, mobile, desktop)
  • Gratis - no requiere credenciales de API pagada

  • Works offline - genera URL sin necesidad de conexión
  • Es un "URL Scheme" - no es una API real

Reglas de Oro al Usar APIs

✅ SIEMPRE HAZ ESTO

  • Usa try/catch para manejar errores
  • Valida datos antes de enviar
  • Guarda credenciales en .env, nunca en código
  • Verifica condiciones en DELETE/UPDATE (.eq(), .where())
  • Documenta qué hace cada función
  • Testea con datos reales antes de producción

❌ NUNCA HAGAS ESTO

  • Credenciales en el código frontend (visibles a todos)
  • DELETE sin filtros (.delete() sin .eq())
  • UPDATE sin condiciones (actualiza TODO)
  • Confiar en validación frontend solamente
  • Ignorar errores de API
  • Hacer llamadas sin async/await
